## 1. Who we are (one breath)

**Epiow** is the **organization operating system** for Hong Kong enterprises: one login workspace, a modular app catalog (HR / CRM / ops / productivity), with **Employment Ordinance & MPF-native** workflows — not a US HR tool with a locale pack.

**Legal:** Epiow Limited · **Product:** epiow.com · **Org:** EpiowAI

---

## 2. What we are not

| Not this | Why it matters for brand |
|----------|---------------------------|
| Another generic global SaaS | Must feel **HK-grounded + enterprise**, not Silicon Valley template |
| Consumer chat / social | No playful sticker energy |
| Pure payroll vendor | Platform OS first; payroll is a module, not the whole face |
| AI-hype wrapper | Intelligence is capability, not the brand costume |
| Four-square “app grid” placeholder | Already retired; never returns |

## 3. Audience (who must trust us in 3 seconds)

| Priority | Who | What they need to feel |
|----------|-----|------------------------|
| P0 | HK SME / mid-market **ops, HR, founders** | “可靠、合規、現代、唔會亂” |
| P1 | IT / implementers | Modular, serious platform (not a toy) |
| P2 | Partners / talent / portfolio visitors | Clear company identity, not a side project |

Buying is **rational + trust**. Logo must not “sell vibes”; it must **look like a system they can run the company on**.

## 4. Positioning line (locked)

**English:** Your Organization's Operating System  
**Chinese (product):** 香港首個專為本地勞工法例設計的模組化商業應用平台  

**Brand job of the line:** “OS for the org” = single place, modular apps, runs the business.  
**Logo job is NOT to illustrate MPF or leave forms.** (Classic rule: mark shows *who*, product shows *what*.)

| # | Attribute | Means in design | Fails if… |
|---|-----------|-----------------|-----------|

| 2 | **Modular** | System feel: consistent modules, lockups, tokens | One-off sticker that can’t scale to favicon |

## 8. Name strategy

| Fact | Implication |
|------|-------------|
| **Epiow** is short, invented, distinctive | **Wordmark can carry the brand** — do not hide the name behind an abstract blob |
| Pronunciation is not obvious to everyone | Mark should not require knowing the story; name must be visible on web |
| “E” alone is overcrowded globally | Pure generic E monogram is weak unless **highly custom** and always next to wordmark |

**Working name treatment:**  
- Logotype always spells **Epiow** (capital E only; rest lowercase or small-caps system TBD).  
- Never “EPIOW” all-caps scream unless technical label.  
- Tagline optional under wordmark on marketing only: *Your Organization's Operating System* — not part of the core mark.

## 9. Logo system architecture (decision)

### Recommended primary structure: **Wordmark-led combination**

```
[ optional mark ]  Epiow
                   ─────────  (clear space rules)
```

| Layer | Role | Priority |
|-------|------|----------|
| **A. Wordmark** | Primary company face (navbar, decks, docs) | **#1 ship** |
| **B. App mark** | Favicon, PWA, social chip, GitHub avatar | #2 derived from A |
| **C. Lockup** | Marketing hero, OG, email | A+B together |
| **D. Mono / reverse** | Print, dark UI, one-color | Required for all of A–C |

**Why not icon-first (what we tried):**  
Early B2B brands pay more for **name recognition**. Icon-only indigo E’s read as “another SaaS tile,” not “Epiow the org OS.”

## 10. Creative directions (only 3 — pick later, design after)

Each direction is **one idea**. No stacking.

### Direction W — **Wordmark OS** (default recommendation)

- Custom lettering for **Epiow** with optical rhythm (not Inter/Roboto dump).  
- Optional tiny system cue in type only (e.g. slightly modular joins, open apertures) — **no orbit illustration**.  
- App icon: first letter or monogram **extracted from the same letter construction**.  

**Says:** “This is a real product company with a name.”  
**Risk if weak:** looks like generic tech type.  
**Mitigation:** distinctive E–p–i rhythm, custom ‘e’/‘o’ counters.

### Direction M — **Modular monogram** (secondary)

- One geometric mark = **modularity of the app catalog** (structure, not hamburger menu).  
- Always ships with wordmark; never orphaned as the only brand.  
- Black-and-white first; must win 16px.

**Says:** “Platform of parts that form a whole.”  
**Risk:** looks like UI chrome or Slack clone.  
**Kill test:** if it reads as menu icon → dead.

### Direction G — **Gateway / single opening** (optional third)

- One abstract portal / aperture = **single OS entry** for the organization.  
- Extreme reduction (Vercel-level courage).  
- Wordmark still primary on web.

**Says:** “One place the org runs.”  
**Risk:** generic arch/n-shape.  
**Kill test:** if it could be any VPN/security brand → dead.

**Not pursuing (archive):** dual orbits, hex tech, four-grid, EO corporate seal, maze-E, AI glow.

## 13. Success criteria (when we say “done”)

1. Someone unfamiliar sees navbar → can **read “Epiow”** and feel **B2B platform**, not consumer app.  
2. Favicon still **legible** and related to wordmark DNA.  
3. On a wall with 飛書 / Linear / Rippling / random indigo E apps → **does not disappear**.  
4. One-sentence rationale works without apology:  
   *“Wordmark-led organization OS: clear name, modular system, grounded for HK enterprise.”*  
5. Scores ≥14/20 on the logo scorecard in `docs/logo-decision.md` (link when updated).
